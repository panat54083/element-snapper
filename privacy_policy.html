Privacy Policy

Element Snapper — Last updated: 2026-02-02

Element Snapper is a browser extension that enables users to capture screenshots of page elements, the visible viewport, or full pages. This policy explains what data (if any) the extension accesses, how it is used, and how users can control or remove it.

Summary

All processing is performed locally in the user's browser. Element Snapper does not collect, transmit, or store user data on external servers. The extension only reads minimal page metadata necessary to perform captures and to generate descriptive filenames.

What data the extension accesses

  • DOM geometry and metadata: element bounding rectangles, page/viewport dimensions, scroll offsets, and a short snippet of element text (up to ~50 characters) used only for optional filename metadata.
  • Screenshots: image data is generated locally in the browser and either saved to the user's Downloads folder or copied to the clipboard, depending on the user's choice.

Why this access is needed

The extension needs to read element geometry and viewport information to:

  • accurately crop screenshots to the selected element (including on high-DPI displays),
  • perform multi-tile stitching for elements larger than the viewport, and
  • optionally include a short, user-friendly filename derived from the element's tag/id/text.

No external transmission

Element Snapper does not make network requests to third-party servers for capture, processing, analytics, or telemetry. No screenshots, DOM content, or metadata are uploaded or shared externally by the extension.

Permissions

The extension requests only the permissions necessary to function:

  • activeTab, tabs: to capture the visible tab and to identify the active tab when the user initiates a capture.
  • downloads: to save screenshots to the user's Downloads folder when the user chooses to download.
  • storage: to persist user settings (format, quality, delay, full-capture, debug, clipboard) locally in the browser.
  • clipboardWrite: to copy image data to the system clipboard when the user enables that option.
  • Content scripts are declared for <all_urls> so users can visually select elements on pages; these scripts only run to provide selection UI and to measure/scroll elements.

Cross-origin and restricted pages

The extension cannot access content inside cross-origin iframes due to browser security restrictions; in such cases the iframe element itself will be captured. The extension also cannot run on browser-internal pages such as chrome:// pages, the Chrome Web Store, or certain privileged URLs.

User control

Users may disable or uninstall the extension at any time via Chrome's Extensions page. Local settings are stored in Chrome's extension storage and will be removed when the extension is uninstalled. Saved screenshots are written to the user's device (Downloads folder) or copied to the clipboard; the extension does not retain copies on external servers.

Changes to this policy
